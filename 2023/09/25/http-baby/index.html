HTTP_baby

开题后看题，发现题目给出php代码如下

<?php
    highlight_file(__FILE__);
    error_reporting(0);

    if ( $_GET['start'] === "op" && $_GET['op']['run'] === "good~~" ) {
        $flag1 = true;
    } else {
        die("You don't understand the GET param");
    }

    if ( $_POST['start'] === "op" && $_POST['op']['run'] === "good~~" ) {
        $flag2 = true;
    } else {
        die("You don't understand the POST param");    
    }

    if ($_COOKIE["role"] === "op") {
        $flag3 = true;
    } else {
        die("You are not op");
    }

    if ($_SERVER["HTTP_X_OP_X"] === "good~~") {
        $flag4 = true;
    } else {
        die("You don't understand HTTP HEADER");
    }

    if ($flag1 && $flag2 && $flag3 && $flag4) {
        echo file_get_contents("/flag");
    }

分析题目

可以发现本题源码中有一个flag文件，只有当flag1234都为真时才能输出flag，则我们需要满足上面http各项要求，那我们就需要Python访问容器，提交相应的get,post,cookie,header,以下是相应的Python脚本

import requests

# 定义目标URL
url = 'http://localhost:51764/'  # 将URL替换为您的实际目标URL

# 构建GET参数、POST参数、Cookie和自定义HTTP头
params = {
    'start': 'op',
    'op[run]': 'good~~'
}
data = {
    'start': 'op',
    'op[run]': 'good~~'
}
cookies = {
    'role': 'op'
}
headers = {
    'X-OP-X': 'good~~'  # 自定义HTTP头
}

# 发送GET请求
response = requests.post(url, params=params, data=data, cookies=cookies, headers=headers)

# 打印响应内容
print(response.text)

requests.post() 方法用于发送 HTTP POST 请求，并接受多个参数，以便进行请求的定制。以下是 requests.post() 方法可接受的一些主要参数：
url (必需)：要发送 POST 请求的目标 URL。
data：要发送的数据，通常是一个字典或字节流。这个参数用于将表单数据发送到服务器。
json：要发送的 JSON 数据，通常用于发送 JSON 格式的数据。如果指定了 json 参数，data 参数将被忽略。
headers：自定义请求头的字典，可以包含额外的请求头信息。
files：用于上传文件的字典，允许你上传文件到服务器。
auth：用于 HTTP 基本身份验证的元组 (username, password)。
timeout：设置请求的超时时间，可以是一个浮点数，单位为秒。
proxies：用于设置代理服务器的字典，以便请求通过代理服务器发送。
verify：设置是否验证 SSL 证书，默认为 True。可以设置为 False 来禁用 SSL 证书验证。
cert：用于指定客户端证书的路径，通常是一个元组 (cert_file, key_file)。
stream：设置是否延迟下载响应内容。如果设置为 True，则只有在访问响应内容时才会下载，适用于大文件下载。
allow_redirects：设置是否允许自动重定向，默认为 True。
params：一个字典，包含 URL 查询参数。
cookies：要发送的 Cookie 数据，可以是一个字典。

参考文章来自菜鸟教程
值得一提的是： Header中一般不允许下划线，下划线与-替换，也就是说Header头中的内容与我们发送的不是一一对应的，将其更改为X-OP-X。
